# Anicca の YouTube 体験（仕様ベース）

**目的:** 夜更かしで YouTube を見ているユーザーに、YouTube 上で Anicca が届く体験を定義する。具体例で「どう見えるか・何が起きるか」を書く。

---

## 1. 誰の体験か

| 項目 | 内容 |
|------|------|
| ペルソナ | 6〜7年習慣化に失敗し続けている 25〜35 歳。夜更かしで「また3時だ…やめなきゃ」と感じている人。 |
| オンボーディングで分かっていること | 「夜更かしして何をしてることが多い？」→ **YouTube** を選択。必要なら「なぜ夜更かししてしまう？」も回答済み（ストレス解消 / 習慣になってる 等）。 |
| 名前 | アプリ内で登録した名前（例: 大祐）を Anicca は持っている。 |

---

## 2. いつ・どこで起きるか

| 項目 | 内容 |
|------|------|
| タイミング | 深夜（例: 0:00〜3:00）。ユーザーが実際に YouTube を見ている時間帯。 |
| 場所 | **YouTube 上**。PC でもスマホでも、YouTube の広告枠として表示される。 |
| 具体的な画面 | YouTube Shorts を縦スクロールしているとき、または動画の前後・スキップ可能なインストリーム広告として。 |

---

## 3. ユーザーに何が見えるか（1つの例）

仕様にある「例1: YouTube広告」をそのまま再掲する。

```
【状況】
深夜0:30、ユーザーはPCでYouTube Shortsを見ている
Aniccaはユーザーの「夜更かし」問題と「YouTubeを見がち」という選択を知っている

【YouTube広告として表示】
┌─────────────────────────────────────┐
│                                     │
│     🌙 大祐、まだ起きてるの？        │
│                                     │
│     もう0:30だよ。                  │
│     明日の自分、泣くよ。            │
│                                     │
│     [今すぐ寝る] [Aniccaを開く]      │
│                                     │
│     広告 • Anicca                   │
│                                     │
└─────────────────────────────────────┘
```

**ポイント**

- **名前で呼ぶ**: オンボーディングで登録した名前（大祐）を表示。他人事じゃない感を出す。
- **時刻を入れる**: 「もう0:30だよ」で、今この瞬間に向けたメッセージだと分かる。
- **刺さる一文**: 「明日の自分、泣くよ。」で、続けることの代償を短く伝える。
- **CTA**: 「今すぐ寝る」で行動を促す。「Aniccaを開く」でアプリに誘導する。

---

## 4. 具体例（シナリオで追う）

### 例 A: 名前が「大祐」のユーザー、0:30 に Shorts を見ている

1. ユーザーは YouTube Shorts を縦スクロールしている。次の Short の前に **5〜6秒の広告** が入る。
2. その広告が Anicca のもの。  
   **見出し:** 「🌙 大祐、まだ起きてるの？」  
   **本文:** 「もう0:30だよ。明日の自分、泣くよ。」  
   **ボタン:** [今すぐ寝る] [Aniccaを開く]
3. ユーザーは「あ、またやってた」と我に返る。  
   - [今すぐ寝る] を押す → 広告が閉じる。YouTube を閉じて寝るかはユーザー次第。  
   - [Aniccaを開く] を押す → アプリが開き、Nudge や習慣画面に誘導できる。
4. **体験の結果**: 「夜更かしの瞬間に、YouTube 上で Anicca が声をかけてくれた」→ 我に返ってスマホを置ける可能性が上がる（Aroadmap の表のとおり）。

### 例 B: 名前が「Sarah」のユーザー、英語圏、1:15 に長尺動画の前広告

1. 動画の前に **スキップ可能なインストリーム広告** が流れる。
2. 表示例（英語）:  
   **「Sarah, still up? It’s 1:15. Tomorrow-you will thank you for stopping now.»**  
   **[Go to bed] [Open Anicca]**
3. スキップするか、5秒見て「まだ起きてるの？」と気づくか、CTA で寝る／Anicca を開くかを選ぶ。

### 例 C: オンボーディングで「夜更かしは SNS」だけ選んで「YouTube」は選んでいない場合

- 現状の仕様では、**夜更かしで何をしているか** で YouTube を選んだ人に YouTube 広告を出す想定。
- 「SNS だけ」の人は、同じ夜更かし Nudge を **別チャネル**（例: アプリ内通知、将来的に他プラットフォーム）で届ける設計にするか、YouTube 広告は出さないかは仕様で決める。

## 5. 体験の流れ（時系列）

| 順番 | 何が起きるか |
|------|----------------|
| 1 | ユーザーがオンボーディングで「夜更かしして何をしてることが多い？」に **YouTube** を選択する。 |
| 2 | Anicca がそのユーザーを「夜更かし × YouTube」として認識し、YouTube 広告ターゲットの候補にする。 |
| 3 | 深夜、ユーザーが YouTube を利用している。広告配信側（Google/YouTube）がそのユーザーに広告枠を出す。 |
| 4 | Anicca の広告がその枠に表示される。文言は名前・時刻・短いメッセージ・CTA。 |
| 5 | ユーザーが広告を見る → 「まだ起きてるの？」と気づく → [今すぐ寝る] または [Aniccaを開く] を選ぶ。 |
| 6 | **体験のゴール**: 我に返って、スマホを置く（または Anicca を開いて次の Nudge に繋げる）。 |

---

## 6. どこで差し替え・拡張するか（例）

| 要素 | 例 |
|------|-----|
| 名前 | オンボーディングの名前フィールドをそのまま使う。 |
| 時刻 | 広告表示時のサーバー時刻または端末時刻。「もう0:30だよ」「It's 1:15」など。 |
| 文言 | A/B テスト用に「明日の自分、泣くよ」「明日の自分が泣くよ」「Tomorrow-you will thank you」など複数パターンを用意できる。 |
| CTA | 「今すぐ寝る」「Aniccaを開く」のほか、「あと5分で寝る」など中間 CTA も将来追加可能。 |
| 表示形式 | Shorts 前の短尺バナー、インストリーム（スキップ可能）、ディスカバリー広告など、YouTube の広告フォーマットに合わせる。 |

---

## 7. ビジネス・制約（仕様メモ）

| 項目 | 内容 |
|------|------|
| 有料限定の可能性 | Aroadmap では「YouTube広告で届く（有料ユーザー限定機能として）」と記載。実装時に有料/無料の線引きを決める。 |
| 広告主 | Anicca（アプリ）。YouTube の広告管理画面でキャンペーンを作成し、ターゲットを「Anicca ユーザーかつ夜更かし×YouTube」などに絞る。 |
| 計測 | インプレッション・クリック（今すぐ寝る / Aniccaを開く）・アプリ遷移を計測し、効果を見る。 |

---

## 8. まとめ（一言で言うと）

**YouTube 体験 = 夜更かしで YouTube を見ている瞬間に、YouTube の広告として「名前＋時刻＋短い一言＋CTA」が表示され、我に返ってスマホを置く（または Anicca を開く）までを Anicca が支援する体験。**
