## 1. アーキテクチャ決定記録 (ADR)

### ADR-001: Workflows vs Agents

**決定**: Workflow優先アーキテクチャを採用

| 基準 | Workflow | Agent | Anicca選択 |
|------|----------|-------|------------|
| **予測可能性** | 高（事前定義パス） | 低（動的判断） | ✅ Workflow |
| **レイテンシ** | 低 | 高 | ✅ Workflow |
| **コスト** | 低 | 高 | ✅ Workflow |
| **デバッグ容易性** | 高 | 低 | ✅ Workflow |
| **柔軟性** | 低 | 高 | Agentが有利だが不要 |

**根拠**: Anthropic "Building Effective Agents" より「可能な限りシンプルな解決策から始める」

**例外**: 以下の場合のみAgent使用
- 苦しみ投稿への返信生成（文脈依存性が高い）
- レビュー返信生成（顧客対応の繊細さ）

### ADR-002: Cron vs Heartbeat

**決定**: タスク種別に応じて使い分け

| タスク種別 | 使用パターン | 理由 |
|-----------|-------------|------|
| **x-poster** | Cron (isolated) | 正確なタイミング必須、文脈不要 |
| **tiktok-poster** | Cron (isolated) | 同上 |
| **trend-hunter** | Cron (interval) | 4時間間隔、独立実行 |
| **feedback-fetch** | Cron (interval) | 4時間間隔、独立実行（**v1.6.3スコープ**） |
| **suffering-detector** | Heartbeat | 最新の会話文脈が必要 |
| **moltbook-responder** | Heartbeat | 会話継続性が必要（**v1.6.3スコープ**） |

**Moltbot研究からの判断基準**:
- 正確なタイミングが必要 → Cron (isolated)
- 会話文脈が必要 → Heartbeat (main session)
- 異なるモデル/設定が必要 → Cron (isolated)
- チャンネル配信が必要 → Cron with delivery

### ADR-003: メモリアーキテクチャ

**決定**: memU 3層階層を採用

```
memory/
├── categories/              ← Category Layer（要約・統合）
│   ├── preferences/
│   │   ├── morning_nudges.md
│   │   └── tone_preferences.md
│   └── behaviors/
│       ├── engagement_patterns.md
│       └── timing_responses.md
├── items/                   ← Item Layer（細粒度メモリ）
│   ├── item_001.json       # "strict tone at 6am gets thumbs_up"
│   └── item_002.json       # "gentle tone works for anxiety"
└── resources/              ← Resource Layer（生データ）
    ├── nudge_feedback_123.json
    └── behavior_event_456.json
```

**コスト削減効果**:
- 現在: 毎回全履歴をプロンプトに含める → 100%コスト
- memU後: Category → Item → Resource の階層的検索 → 10-20%コスト

#### Prisma Schema (memU テーブル)

> **実装時の適応規約**:
> - **FK参照先**: `userId` は `String @db.Uuid` として定義。memUテーブルはユーザーテーブルとの直接FK制約を設けずアプリ層で整合性を担保（既存の `UserSubscription.userId` と同じ形式）。**注意**: ops系テーブル（closed-loop-ops/02-data-layer.md）はFK制約あり（REFERENCES + @relation）— 責務が異なるため方針が異なる
> - **型**: `@db.Uuid @default(dbgenerated("gen_random_uuid()"))`, `@db.Timestamptz @default(now())`
> - **命名**: `@map("snake_case")`, `@@map("table_names")`
> - **マイグレーション**: SQL手動適用 + `prisma db pull`（v0.3方針）

```prisma
// apps/api/prisma/schema.prisma に追加
// 実装例（既存規約適用済み）

model MemoryResource {
  id          String   @id @db.Uuid @default(dbgenerated("gen_random_uuid()"))
  userId      String   @db.Uuid @map("user_id")  // memU: FK制約なし（アプリ層で整合性担保）。ops系テーブルはFK制約あり（02-data-layer.md参照）
  type        String   @db.VarChar(50) @map("type")
  content     Json     @db.JsonB @map("content")
  metadata    Json?    @db.JsonB @map("metadata")
  createdAt   DateTime @db.Timestamptz @default(now()) @map("created_at")

  items       MemoryItem[]

  @@index([userId, type])
  @@index([createdAt])
  @@map("memory_resources")
}

model MemoryItem {
  id           String   @id @db.Uuid @default(dbgenerated("gen_random_uuid()"))
  userId       String   @db.Uuid @map("user_id")
  resourceId   String   @db.Uuid @map("resource_id")
  content      String   @map("content")
  embedding    Bytes?   @map("embedding")  // text-embedding-3-small (1536 dims)
  categoryId   String?  @db.Uuid @map("category_id")
  relevance    Float    @default(1.0) @map("relevance")
  createdAt    DateTime @db.Timestamptz @default(now()) @map("created_at")
  updatedAt    DateTime @db.Timestamptz @updatedAt @map("updated_at")

  resource     MemoryResource @relation(fields: [resourceId], references: [id], onDelete: Cascade)
  category     MemoryCategory? @relation(fields: [categoryId], references: [id])

  @@index([userId, categoryId])
  @@index([userId, createdAt])
  @@map("memory_items")
}

model MemoryCategory {
  id               String    @id @db.Uuid @default(dbgenerated("gen_random_uuid()"))
  userId           String    @db.Uuid @map("user_id")
  name             String    @db.VarChar(255) @map("name")
  summary          String    @map("summary")
  itemCount        Int       @default(0) @map("item_count")
  lastSummarizedAt DateTime? @db.Timestamptz @map("last_summarized_at")
  createdAt        DateTime  @db.Timestamptz @default(now()) @map("created_at")
  updatedAt        DateTime  @db.Timestamptz @updatedAt @map("updated_at")

  items            MemoryItem[]

  @@unique([userId, name])
  @@index([userId])
  @@map("memory_categories")
}
```

**インデックス戦略**:

| インデックス | 用途 |
|-------------|------|
| `userId, type` on Resource | ユーザー別リソース検索 |
| `userId, categoryId` on Item | カテゴリ別アイテム検索 |
| `userId, createdAt` on Item | 時系列検索（Discounted TS用） |

### 5.1 ML送信時間最適化 (STO)

```javascript
/**
 * Send Time Optimization
 * 
 * Based on:
 * - Last 60 days of engagement data
 * - Hourly aggregation, localized to timezone
 * - Weekly model refresh
 */

// apps/api/src/services/sendTimeOptimization.js

/**
 * Calculate optimal send time for user
 * 
 * @param {string} userId - User ID
 * @param {string} problemType - Problem type
 * @returns {number} - Optimal hour (0-23) in user's timezone
 */
async function getOptimalSendTime(userId, problemType) {
  // 1. Get user's engagement history (last 60 days)
  const history = await prisma.nudgeFeedback.findMany({
    where: {
      userId,
      problemType,
      createdAt: {
        gte: new Date(Date.now() - 60 * 24 * 60 * 60 * 1000),
      },
    },
    select: {
      scheduledHour: true,
      outcome: true, // 'tapped' | 'ignored' | 'thumbs_up' | 'thumbs_down'
      createdAt: true,
    },
  });
  
  if (history.length < 10) {
    // Insufficient data: use population average
    return getPopulationOptimalHour(problemType);
  }
  
  // 2. Aggregate by hour
  const hourlyStats = {};
  for (let h = 0; h < 24; h++) {
    hourlyStats[h] = { positive: 0, total: 0 };
  }
  
  for (const record of history) {
    const hour = record.scheduledHour;
    hourlyStats[hour].total++;
    if (['tapped', 'thumbs_up'].includes(record.outcome)) {
      hourlyStats[hour].positive++;
    }
  }
  
  // 3. Calculate engagement rate per hour
  let bestHour = 8; // Default
  let bestRate = 0;
  
  for (const [hour, stats] of Object.entries(hourlyStats)) {
    if (stats.total >= 3) { // Minimum samples
      const rate = stats.positive / stats.total;
      if (rate > bestRate) {
        bestRate = rate;
        bestHour = parseInt(hour);
      }
    }
  }
  
  // 4. Avoid inconvenient times (11pm - 6am)
  if (bestHour >= 23 || bestHour < 6) {
    // Shift to nearest acceptable time
    bestHour = bestHour >= 23 ? 22 : 7;
  }
  
  return bestHour;
}

/**
 * Get population-level optimal hour for problem type
 */
async function getPopulationOptimalHour(problemType) {
  const defaults = {
    staying_up_late: 22,    // Evening reminder
    cant_wake_up: 6,        // Morning motivation
    self_loathing: 20,      // Evening reflection
    rumination: 14,         // Afternoon break
    procrastination: 9,     // Morning kickstart
    anxiety: 7,             // Morning grounding
    loneliness: 19,         // Evening connection
    anger: 12,              // Midday pause
    // ... other types
  };
  
  return defaults[problemType] || 9; // Default to 9am
}

module.exports = { getOptimalSendTime, refreshWeeklyModel, getHourlyEngagement };

/**
 * Weekly Model Refresh (週次更新)
 *
 * Cron: 毎週日曜 03:00 JST
 * 保存先: UserStoModel テーブル
 *
 * @param {string} userId - User ID
 * @returns {Object} - Updated hourly engagement data
 */
async function refreshWeeklyModel(userId) {
  // 過去7日間のデータを集計
  const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000);

  const recentFeedback = await prisma.nudgeFeedback.findMany({
    where: {
      userId,
      createdAt: { gte: sevenDaysAgo }
    },
    select: {
      scheduledHour: true,
      outcome: true
    }
  });

  // hourlyEngagement 計算
  const hourlyEngagement = {};
  for (let h = 0; h < 24; h++) {
    hourlyEngagement[h] = { positive: 0, total: 0, rate: 0 };
  }

  for (const fb of recentFeedback) {
    const h = fb.scheduledHour;
    hourlyEngagement[h].total++;
    if (['tapped', 'thumbs_up'].includes(fb.outcome)) {
      hourlyEngagement[h].positive++;
    }
  }

  // engagement rate 計算
  for (let h = 0; h < 24; h++) {
    const stats = hourlyEngagement[h];
    stats.rate = stats.total >= 3 ? stats.positive / stats.total : 0;
  }

  // UserStoModel に保存（upsert）
  await prisma.userStoModel.upsert({
    where: { userId },
    create: {
      userId,
      hourlyEngagement: JSON.stringify(hourlyEngagement),
      lastRefreshedAt: new Date()
    },
    update: {
      hourlyEngagement: JSON.stringify(hourlyEngagement),
      lastRefreshedAt: new Date()
    }
  });

  return hourlyEngagement;
}

/**
 * Get cached hourly engagement data
 *
 * @param {string} userId - User ID
 * @returns {Object|null} - Cached hourly engagement or null
 */
async function getHourlyEngagement(userId) {
  const model = await prisma.userStoModel.findUnique({
    where: { userId },
    select: { hourlyEngagement: true, lastRefreshedAt: true }
  });

  if (!model) return null;

  // 7日以上古い場合は再計算を推奨
  const daysSinceRefresh = (Date.now() - model.lastRefreshedAt.getTime()) / (1000 * 60 * 60 * 24);

  return {
    data: JSON.parse(model.hourlyEngagement),
    stale: daysSinceRefresh > 7
  };
}
```

```javascript
  // 0. Check cool-off since last send (直前送信からの最小間隔)
  const lastSend = await prisma.nudgeSent.findFirst({
    where: { userId },
    orderBy: { createdAt: 'desc' }
  });

  if (lastSend) {
    const minutesSince = (Date.now() - lastSend.createdAt.getTime()) / (1000 * 60);
    if (minutesSince < FATIGUE_CONFIG.min_minutes_since_last_send) {
      return { allowed: false, reason: 'cooloff_since_last_send' };
    }
  }

  // 1. Check daily limit
  const today = new Date();
  today.setHours(0, 0, 0, 0);

  const todayCount = await prisma.nudgeSent.count({
    where: {
      userId,
      createdAt: { gte: today },
    },
  });

  if (todayCount >= FATIGUE_CONFIG.daily_limit_total) {
    return { allowed: false, reason: 'daily_limit_reached' };
  }

  // 2. Check problem type limit
  const typeCount = await prisma.nudgeSent.count({
    where: {
      userId,
      problemType,
      createdAt: { gte: today },
    },
  });
  
  if (typeCount >= FATIGUE_CONFIG.daily_limit_per_type) {
    return { allowed: false, reason: 'type_limit_reached' };
  }
  
  // 3. Check cool-off after negative feedback
  const lastNegative = await prisma.nudgeFeedback.findFirst({
    where: {
      userId,
      outcome: 'thumbs_down',
    },
    orderBy: { createdAt: 'desc' },
  });
  
  if (lastNegative) {
    const hoursSince = (Date.now() - lastNegative.createdAt.getTime()) / (1000 * 60 * 60);
    if (hoursSince < FATIGUE_CONFIG.cooloff_hours_after_negative) {
      return { allowed: false, reason: 'cooloff_active' };
    }
  }
  
  // 4. Check hook reuse
  if (hookId) {
    const lastSameHook = await prisma.nudgeSent.findFirst({
      where: {
        userId,
        hookId,
      },
      orderBy: { createdAt: 'desc' },
    });
    
    if (lastSameHook) {
      const hoursSince = (Date.now() - lastSameHook.createdAt.getTime()) / (1000 * 60 * 60);
      if (hoursSince < FATIGUE_CONFIG.min_hours_between_same_hook) {
        return { allowed: false, reason: 'hook_recently_used' };
      }
    }
  }
  
  return { allowed: true };
}
```

```javascript
/**
 * 入力検証エラー（A6-A8テスト対応）
 */
class InvalidInputError extends Error {
  constructor(message) {
    super(message);
    this.name = 'InvalidInputError';
  }
}

/**
 * Dynamic Prior Thompson Sampling でhookを選択
 *
 * @param {Array<Object>} hooks - Hook候補リスト
 * @param {Object} options - オプション
 * @param {number} options.epsilon - 探索率 (default: 0.1)
 * @param {number} options.priorStrength - Prior強度 (default: 0.1)
 * @param {number} options.discountFactor - 減衰係数 (default: 0.95/週)
 * @param {Function} options.rng - 乱数生成関数（テスト用、default: Math.random）
 * @param {Function} options.clock - 現在時刻関数（テスト用、default: Date.now）
 * @returns {Object} 選択されたhook
 * @throws {InvalidInputError} hooks が空、非配列、または createdAt 欠落の場合
 */
async function selectHookThompson(hooks, options = {}) {
  const {
    epsilon = 0.1,
    priorStrength = 0.1,
    discountFactor = 0.95,
    rng = Math.random,  // テスト用注入可能
    clock = Date.now    // テスト用注入可能
  } = options;

  // A7: 非配列検証
  if (!Array.isArray(hooks)) {
    throw new InvalidInputError('hooks must be an array');
  }

  // A6: 空配列検証
  if (hooks.length === 0) {
    throw new InvalidInputError('hooks must be a non-empty array');
  }

  // A8: createdAt必須検証
  for (const hook of hooks) {
    if (!hook.createdAt) {
      throw new InvalidInputError(`hook "${hook.id || 'unknown'}" is missing required field: createdAt`);
    }
  }

  // 各hookに対してDynamic Prior Thompson Samplingスコアを計算
  const scored = hooks.map(hook => ({
    hook,
    score: calculateDynamicPriorTS(hook, { epsilon, priorStrength, discountFactor, rng, clock })
  }));

  // 最高スコアのhookを選択
  return scored.reduce((best, current) =>
    current.score > best.score ? current : best
  ).hook;
}

/**
 * Dynamic Prior Thompson Samplingスコア計算
 *
 * @param {Object} hook - Hook オブジェクト
 * @param {Object} params - パラメータ（rng/clock含む）
 * @returns {number} Thompson Samplingスコア (0-1)
 */
function calculateDynamicPriorTS(hook, { epsilon, priorStrength, discountFactor, rng, clock }) {
  const { successCount = 0, failureCount = 0, createdAt } = hook;

  // 時間減衰（週単位）- clock注入対応
  const weeksSinceCreation = (clock() - new Date(createdAt).getTime()) / (7 * 24 * 60 * 60 * 1000);
  const decay = Math.pow(discountFactor, weeksSinceCreation);

  // 減衰したカウント
  const alpha = 1 + (successCount * decay);
  const beta = 1 + (failureCount * decay);

  // Beta分布からサンプリング - rng注入対応
  const sample = betaSample(alpha, beta, rng);

  // Dynamic Prior: ε確率で探索、(1-ε)確率で活用
  if (rng() < epsilon) {
    return rng(); // 探索: ランダムスコア
  }
  return sample; // 活用: Thompson Samplingスコア
}

/**
 * Beta分布からのサンプリング（Jöhnk's algorithm）
 * @param {number} alpha - α パラメータ (> 0)
 * @param {number} beta - β パラメータ (> 0)
 * @param {Function} rng - 乱数生成関数（テスト用注入）
 * @returns {number} 0-1 の範囲のサンプル値
 */
function betaSample(alpha, beta, rng = Math.random) {
  // Jöhnk's algorithm for Beta distribution
  while (true) {
    const u1 = rng();
    const u2 = rng();
    const x = Math.pow(u1, 1 / alpha);
    const y = Math.pow(u2, 1 / beta);
    if (x + y <= 1) {
      return x / (x + y);
    }
  }
}
```

```javascript
// apps/api/src/services/verifier.js

const { writeToDLQ } = require('./errorHandler');

/**
 * コンテンツ生成＋検証ループ（再生成対応）
 *
 * @param {Function} generateFn - コンテンツ生成関数 (feedback) => Promise<string>
 * @param {Object} options - オプション
 * @param {number} options.threshold - 合格閾値 (default: 3)
 * @param {number} options.maxRetries - 最大再生成回数 (default: 3)
 * @param {string} options.skillName - スキル名（DLQ記録用）
 * @param {Object} options.context - コンテキスト（DLQ記録用）
 * @returns {Object} { passed: boolean, content: string, score: number, attempts: number }
 */
async function verifyWithRegeneration(generateFn, options = {}) {
  const { threshold = 3, maxRetries = 3, skillName = 'unknown', context = {} } = options;

  let bestAttempt = null;
  let attempts = [];

  for (let i = 0; i < maxRetries; i++) {
    // 生成（前回フィードバックを渡す）
    const feedback = attempts.length > 0 ? attempts[attempts.length - 1].feedback : null;
    const content = await generateFn(feedback);

    // 検証
    const result = await verifyContent(content);
    attempts.push({ content, ...result, attempt: i + 1 });

    // 合格判定
    if (result.score >= threshold) {
      return { passed: true, content, score: result.score, attempts: i + 1 };
    }

    // best_attempt更新（最高スコアを記録）
    if (!bestAttempt || result.score > bestAttempt.score) {
      bestAttempt = { content, score: result.score, attempt: i + 1 };
    }
  }

  // 全試行失敗 → DLQ記録 + best_attempt返却
  await writeToDLQ(skillName, new Error(`Verification failed after ${maxRetries} attempts`), {
    ...context,
    attempts,
    bestAttempt
  });

  return {
    passed: false,
    content: bestAttempt.content,
    score: bestAttempt.score,
    attempts: maxRetries,
    usedBestAttempt: true
  };
}

 * 単一コンテンツ品質検証

    const response = await anthropic.messages.create({
      model,
```

```javascript
/**
 * エラーハンドリング（Equal Jitter Backoff + DLQ + 401アラート）
 *
 * @param {Function} fn - 実行する関数
 * @param {Object} options - オプション
 * @param {number} options.maxRetries - 最大リトライ回数 (default: 3)
 * @param {number} options.baseDelay - 基本遅延ms (default: 1000)
 * @param {string} options.skillName - スキル名（DLQ記録用）
 * @returns {any} 関数の戻り値
 * @throws {Error} 全リトライ失敗時
 */
export async function withRetry(fn, options = {}) {
  const { maxRetries = 3, baseDelay = 1000, skillName = 'unknown' } = options;

  let lastError;
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error;

      // 4xx系はリトライしない（401, 403, 404）
      if (error.status >= 400 && error.status < 500 && error.status !== 429) {
        // E3: 401/403時にSlackアラート送信
        if (error.status === 401 || error.status === 403) {
          await sendSlackAlert('#alerts', `[${skillName}] Auth error: ${error.status} - ${error.message}`);
          await writeToDLQ(skillName, error, { ...options.context, alerted: true });
        }
        throw error;
      }

      // Equal Jitter Backoff
      const delay = calculateEqualJitter(baseDelay, attempt);

      // 429の場合はRetry-After尊重
      if (error.status === 429 && error.retryAfter) {
        await sleep(error.retryAfter * 1000);
      } else {
        await sleep(delay);
      }
    }
  }

  // DLQに記録
  await writeToDLQ(skillName, lastError, options.context);
  throw lastError;
}

/**
 * Equal Jitter Backoff計算
 * delay = base * 2^attempt + random(0, base * 2^attempt)
 */
export function calculateEqualJitter(baseDelay, attempt) {
  const expDelay = baseDelay * Math.pow(2, attempt);
  const jitter = Math.random() * expDelay;
  return expDelay + jitter;
}

/**
 * DLQ書き込み
 */
export async function writeToDLQ(skillName, error, context = {}) {
  const entry = {
    timestamp: new Date().toISOString(),
    skill: skillName,
    error: error.message,
    context,
    resolution: null
  };

  const filePath = path.join(DLQ_DIR, `${skillName}.jsonl`);
  await fs.mkdir(DLQ_DIR, { recursive: true });
  await fs.appendFile(filePath, JSON.stringify(entry) + '\n');
}
```

```javascript
// Circuit Breaker状態管理
const CircuitState = {
  CLOSED: 'CLOSED',     // 正常動作
  OPEN: 'OPEN',         // 全リクエスト即座に失敗
  HALF_OPEN: 'HALF_OPEN' // テスト中
};

// サービスごとのCircuit状態
const circuits = new Map();

/**
 * Circuit Breaker付きリトライ
 *
 * @param {string} serviceName - サービス識別子
 * @param {Function} fn - 実行する関数
 * @param {Object} options - オプション
 * @param {number} options.failureThreshold - OPEN移行の失敗回数閾値 (default: 5)
 * @param {number} options.recoveryTimeout - OPEN→HALF_OPEN までのms (default: 30000)
 * @param {number} options.successThreshold - HALF_OPEN→CLOSED への成功回数 (default: 2)
 * @returns {any} 関数の戻り値
 * @throws {Error} Circuit OPEN時またはリトライ失敗時
 */
async function withCircuitBreaker(serviceName, fn, options = {}) {
  const {
    failureThreshold = 5,
    recoveryTimeout = 30000,
    successThreshold = 2,
    ...retryOptions
  } = options;

  // 初期化
  if (!circuits.has(serviceName)) {
    circuits.set(serviceName, {
      state: CircuitState.CLOSED,
      failures: 0,
      successes: 0,
      lastFailure: null,
      nextAttempt: null
    });
  }

  const circuit = circuits.get(serviceName);

  // OPEN状態チェック
  if (circuit.state === CircuitState.OPEN) {
    if (Date.now() < circuit.nextAttempt) {
      throw new Error(`Circuit OPEN for ${serviceName}. Retry after ${new Date(circuit.nextAttempt).toISOString()}`);
    }
    // 復帰タイムアウト経過 → HALF_OPEN
    circuit.state = CircuitState.HALF_OPEN;
    circuit.successes = 0;
  }

  try {
    const result = await withRetry(fn, retryOptions);

    // 成功処理
    if (circuit.state === CircuitState.HALF_OPEN) {
      circuit.successes++;
      if (circuit.successes >= successThreshold) {
        circuit.state = CircuitState.CLOSED;
        circuit.failures = 0;
      }
    } else {
      circuit.failures = 0;
    }

    return result;
  } catch (error) {
    // 失敗処理
    circuit.failures++;
    circuit.lastFailure = Date.now();

    if (circuit.state === CircuitState.HALF_OPEN) {
      // HALF_OPENで失敗 → 即座にOPEN
      circuit.state = CircuitState.OPEN;
      circuit.nextAttempt = Date.now() + recoveryTimeout;
    } else if (circuit.failures >= failureThreshold) {
      // 閾値到達 → OPEN
      circuit.state = CircuitState.OPEN;
      circuit.nextAttempt = Date.now() + recoveryTimeout;
    }

    throw error;
  }
}

/**
 * Circuit状態取得（監視・テスト用）
 */
function getCircuitState(serviceName) {
  return circuits.get(serviceName) || null;
}

/**
 * Circuit状態リセット（テスト用）
 */
function resetCircuit(serviceName) {
  circuits.delete(serviceName);
}

  calculateEqualJitter,
  writeToDLQ,
  sleep,
  withCircuitBreaker,
```

```javascript
// apps/api/src/services/memuService.js

const { prisma } = require('../lib/prisma');
const { callLLM } = require('./llmService');
const { OpenAI } = require('openai');

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

/**
 * Embedding生成（text-embedding-3-small）
 * @param {string} text - 埋め込み対象テキスト
 * @returns {Float32Array} 1536次元のembedding
 */
async function generateEmbedding(text) {
  const response = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: text.slice(0, 8000),  // 最大8000文字
  });
  return new Float32Array(response.data[0].embedding);
}

/**
 * コサイン類似度計算
 * @param {Float32Array} a - ベクトルA
 * @param {Float32Array} b - ベクトルB
 * @returns {number} 類似度 (0-1)
 */
function cosineSimilarity(a, b) {
  if (a.length !== b.length) return 0;
  let dotProduct = 0, normA = 0, normB = 0;
  for (let i = 0; i < a.length; i++) {
    dotProduct += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }
  return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
}

/**
 * Buffer → Float32Array 変換
 * DBから取得したembedding（Buffer型）をFloat32Arrayに復元
 * @param {Buffer} buffer - Prisma から取得した embedding Buffer
 * @returns {Float32Array} 復元された 1536次元ベクトル
 */
function bufferToFloat32Array(buffer) {
  // Buffer を ArrayBuffer に変換し、Float32Array として解釈
  const arrayBuffer = buffer.buffer.slice(
    buffer.byteOffset,
    buffer.byteOffset + buffer.byteLength
  );
  return new Float32Array(arrayBuffer);
}

/**
 * Float32Array → Buffer 変換（memorize保存用）
 * generateEmbeddingの結果をPrisma Bytes型として保存
 * @param {Float32Array} float32Array - 1536次元のembedding
 * @returns {Buffer} Prisma保存用のBuffer
 */
function float32ArrayToBuffer(float32Array) {
  // Float32Array の underlying buffer からコピー
  return Buffer.from(float32Array.buffer);
}

/**
 * Round-trip検証（テスト用）
 * Float32Array → Buffer → Float32Array が一致することを確認
 * @param {Float32Array} original - 元のembedding
 * @returns {boolean} 一致すればtrue
 */
function verifyRoundTrip(original) {
  const buffer = float32ArrayToBuffer(original);
  const restored = bufferToFloat32Array(buffer);
  if (original.length !== restored.length) return false;
  for (let i = 0; i < original.length; i++) {
    if (original[i] !== restored[i]) return false;
  }
  return true;
}

/**
 * Resource から Item を抽出（LLM使用）
 * Mem0 v2.0 の2フェーズパイプラインに基づく
 *
 * @param {Object} content - Resource の content オブジェクト
 * @param {Object} options - オプション
 * @param {number} options.maxItems - 最大Item数 (default: 5)
 * @param {string} options.language - 言語 (default: 'ja')
 * @returns {Array<{content: string, importance: number}>} 抽出されたItem配列
 */
async function extractItemsFromResource(content, options = {}) {
  const { maxItems = 5, language = 'ja' } = options;

  const prompt = `以下のコンテンツから重要な記憶項目を${maxItems}個以内で抽出してください。
各項目は独立して意味を持つ簡潔な文にしてください。

コンテンツ:
${JSON.stringify(content, null, 2)}

出力形式 (JSON配列):
[{"content": "抽出された記憶", "importance": 0-5のスコア}, ...]`;

  const response = await callLLM({
    model: 'gpt-4o-mini',
    messages: [{ role: 'user', content: prompt }],
    response_format: { type: 'json_object' }
  });

  let extracted;
  try {
    const parsed = JSON.parse(response.choices[0].message.content);
    extracted = Array.isArray(parsed) ? parsed : parsed.items || [];
  } catch {
    extracted = [];
  }

  return extracted.slice(0, maxItems);
}

/**
 * CAGキャッシュインターフェース
 * @typedef {Object} CAGCacheInterface
 * @property {function(string): Promise<Object|null>} get - キャッシュ取得
 * @property {function(string, Object, number): Promise<void>} set - キャッシュ設定
 * @property {function(string): Promise<void>} invalidate - キャッシュ無効化
 */

// デフォルトのCAGキャッシュ実装（Map-based、本番はRedis推奨）
const cagCache = {
  _cache: new Map(),
  _ttl: 3600 * 1000, // 1時間

  async get(key) {
    const entry = this._cache.get(key);
    if (!entry) return null;
    if (Date.now() > entry.expiresAt) {
      this._cache.delete(key);
      return null;
    }
    return entry.data;
  },

  async set(key, data, ttl = this._ttl) {
    this._cache.set(key, { data, expiresAt: Date.now() + ttl });
  },

  async invalidate(key) {
    this._cache.delete(key);
  }
};

/**
 * メモリ検索（3段階フォールバック + fallbackUsed追跡）
 * 1. CAG: キャッシュ → 静的データからキーワードマッチ
 * 2. RAG: Embedding ベースの類似検索
 * 3. LLM: LLMに直接クエリ
 *
 * @param {string} userId - ユーザーID
 * @param {string} query - 検索クエリ
 * @param {Object} options - オプション
 * @param {number} options.limit - 最大結果数 (default: 5)
 * @param {'cag'|'rag'|'llm'} options.mode - 検索モード (default: 'cag')
 * @param {CAGCacheInterface} options.cache - 注入可能なキャッシュ（テスト用）
 * @returns {{items: Array<{id, content, score}>, fallbackUsed: boolean, source: 'cag'|'rag'|'llm'}}
 */
async function searchMemory(userId, query, options = {}) {
  const { limit = 5, mode = 'cag', cache = cagCache } = options;

  // CAGモード: キャッシュ → PostgreSQL ILIKE検索
  if (mode === 'cag') {
    // 1. キャッシュチェック
    const cacheKey = `query:${query}:${userId}`;
    const cached = await cache.get(cacheKey);
    if (cached) {
      return { items: cached.items, fallbackUsed: false, source: 'cag' };
    }

    // 2. PostgreSQL検索
    const items = await prisma.memoryItem.findMany({
      where: {
        userId,
        content: { contains: query, mode: 'insensitive' }
      },
      take: limit,
      orderBy: { createdAt: 'desc' }
    });

    if (items.length > 0) {
      const result = items.map(item => ({
        id: item.id,
        content: item.content,
        score: 1.0
      }));
      // キャッシュに保存
      await cache.set(cacheKey, { items: result });
      return { items: result, fallbackUsed: false, source: 'cag' };
    }

    // CAGで見つからない場合、RAGにフォールバック
    const ragResult = await searchMemory(userId, query, { limit, mode: 'rag', cache });
    return { items: ragResult.items, fallbackUsed: true, source: 'rag' };
  }

  // RAGモード: Embedding検索（コサイン類似度）
  if (mode === 'rag') {
    const queryEmbedding = await generateEmbedding(query);
    const allItems = await prisma.memoryItem.findMany({
      where: { userId, embedding: { not: null } },
      select: { id: true, content: true, embedding: true }
    });

    // アプリ層でコサイン類似度計算
    // 重要: DBから取得したembeddingはBuffer型なのでFloat32Arrayに変換
    const scored = allItems.map(item => {
      // Buffer → Float32Array 変換
      const embeddingFloat32 = bufferToFloat32Array(item.embedding);
      return {
        ...item,
        score: cosineSimilarity(queryEmbedding, embeddingFloat32)
      };
    });

    const results = scored
      .sort((a, b) => b.score - a.score)
      .slice(0, limit)
      .map(({ id, content, score }) => ({ id, content, score }));

    // RAG結果0件時にLLMにフォールバック
    if (results.length === 0) {
      const llmResult = await searchMemory(userId, query, { limit, mode: 'llm', cache });
      return { items: llmResult.items, fallbackUsed: true, source: 'llm' };
    }

    return { items: results, fallbackUsed: false, source: 'rag' };
  }

  // LLMモード: LLMに直接クエリ
  if (mode === 'llm') {
    const allItems = await prisma.memoryItem.findMany({
      where: { userId },
      take: 100,
      orderBy: { createdAt: 'desc' }
    });

    if (allItems.length === 0) {
      return { items: [], fallbackUsed: false, source: 'llm' };
    }

    const prompt = `以下の記憶アイテムから「${query}」に関連するものを${limit}件選んでください。

記憶アイテム:
${allItems.map((item, i) => `${i}: ${item.content}`).join('\n')}

出力形式 (JSON配列):
[{"index": 0, "relevance": 0.9}, ...]`;

    const response = await callLLM({
      model: 'gpt-4o-mini',
      messages: [{ role: 'user', content: prompt }],
      response_format: { type: 'json_object' }
    });

    let selected;
    try {
      const parsed = JSON.parse(response.choices[0].message.content);
      selected = Array.isArray(parsed) ? parsed : parsed.results || [];
    } catch {
      selected = [];
    }

    const items = selected
      .filter(s => s.index < allItems.length)
      .slice(0, limit)
      .map(s => ({
        id: allItems[s.index].id,
        content: allItems[s.index].content,
        score: s.relevance || 0.5
      }));

    return { items, fallbackUsed: false, source: 'llm' };
  }

  return { items: [], fallbackUsed: false, source: 'cag' };
}

/**
 * Category自動更新（20 Item超過時）
 * Categoryはnameでユニーク検索し、idはUUIDを使用
 *
 * @param {string} userId - ユーザーID (UUID)
 * @param {string} categoryName - カテゴリ名
 * @returns {Object} 更新結果
 */
async function updateCategoryIfNeeded(userId, categoryName) {
  // Categoryをname + userIdでユニーク検索
  let category = await prisma.memoryCategory.findFirst({
    where: { userId, name: categoryName }
  });

  // 存在しない場合は新規作成
  if (!category) {
    category = await prisma.memoryCategory.create({
      data: { userId, name: categoryName, summary: '', itemCount: 0 }
    });
  }

  // categoryIdでItem数をカウント
  const itemCount = await prisma.memoryItem.count({
    where: { userId, categoryId: category.id }
  });

  if (itemCount <= 20) {
    return { updated: false, itemCount, categoryId: category.id };
  }

  // 20件超過 → LLMで要約してCategory更新
  const items = await prisma.memoryItem.findMany({
    where: { userId, categoryId: category.id },
    orderBy: { createdAt: 'desc' }
  });

  const prompt = `以下の${items.length}件の記憶を300トークン以内で要約してください:

${items.map(i => `- ${i.content}`).join('\n')}`;

  const response = await callLLM({
    model: 'gpt-4o-mini',
    messages: [{ role: 'user', content: prompt }]
  });

  const summary = response.choices[0].message.content;

  // UUIDのidで更新
  await prisma.memoryCategory.update({
    where: { id: category.id },
    data: { summary, itemCount, lastSummarizedAt: new Date() }
  });

  return { updated: true, itemCount, summary, categoryId: category.id };
}

/**
 * 全カテゴリの自動更新チェック（memorize後に呼び出し）
 * @param {string} userId - ユーザーID
 * @returns {string[]} 更新されたカテゴリ名の配列
 */
async function checkAndUpdateAllCategories(userId) {
  const categories = await prisma.memoryCategory.findMany({
    where: { userId }
  });

  const updatedCategories = [];
  for (const cat of categories) {
    const result = await updateCategoryIfNeeded(userId, cat.name);
    if (result.updated) {
      updatedCategories.push(cat.name);
    }
  }

  return updatedCategories;
}
```

```javascript
    return res.status(400).json({ error: parsed.error.issues });
  }

  const { id } = req.params;
  const existing = await prisma.hook.findUnique({ where: { id }, select: { id: true } });
  if (!existing) {
    return res.status(404).json({ error: 'Hook not found' });
  }

  const updateData = {};
  if (parsed.data.outcome === 'success') updateData.successCount = { increment: 1 };
  if (parsed.data.outcome === 'failure') updateData.failureCount = { increment: 1 };
  if (parsed.data.engagement !== undefined) {
    // avgEngagement の更新（移動平均）
    const current = await prisma.hook.findUnique({ where: { id } });
    const posts = await prisma.hookPost.count({ where: { hookId: id } });
    updateData.avgEngagement = posts > 0
      ? ((current.avgEngagement || 0) * posts + parsed.data.engagement) / (posts + 1)
      : parsed.data.engagement;
  }
  updateData.lastUsedAt = new Date();

  const updated = await prisma.hook.update({
    where: { id },
    data: updateData,
    select: { id: true, successCount: true, failureCount: true, avgEngagement: true, updatedAt: true }
  });

  res.json(updated);
});
```

```javascript
// apps/api/src/routes/agent/memory.js
import { Router } from 'express';
import { z } from 'zod';
import { prisma } from '../../lib/prisma.js';
import { requireAgentAuth } from '../../middleware/requireAgentAuth.js';
import {
  extractItemsFromResource,
  generateEmbedding,
  searchMemory,
  checkAndUpdateAllCategories,
  float32ArrayToBuffer
} from '../../services/memuService.js';

const router = Router();
router.use(requireAgentAuth);

const MemorizeSchema = z.object({
  userId: z.string().uuid(),
  resourceType: z.string().min(1).max(50),
  content: z.record(z.unknown()),
  metadata: z.record(z.unknown()).optional()
});

const RetrieveSchema = z.object({
  userId: z.string().uuid(),
  query: z.string().min(1),
  limit: z.number().int().min(1).max(20).default(5),
  mode: z.enum(['cag', 'rag', 'llm']).default('cag')
});

// POST /api/agent/memory/memorize
router.post('/memorize', async (req, res) => {
  const parsed = MemorizeSchema.safeParse(req.body);
  if (!parsed.success) {
    return res.status(400).json({ error: parsed.error.issues });
  }

  const { userId, resourceType, content, metadata } = parsed.data;

  // トランザクションで Resource → Item 作成 → Embedding生成 → Category自動更新
  const result = await prisma.$transaction(async (tx) => {
    const resource = await tx.memoryResource.create({
      data: { userId, type: resourceType, content, metadata }
    });

    // LLMでItem抽出（実装はmemuService.jsに委譲）
    const items = await extractItemsFromResource(content);

    // 各ItemにEmbedding生成して保存
    const createdItems = await Promise.all(
      items.map(async (item) => {
        const embedding = await generateEmbedding(item.content);
        return tx.memoryItem.create({
          data: {
            userId,
            resourceId: resource.id,
            content: item.content,
            embedding: float32ArrayToBuffer(embedding)  // Float32Array→Buffer（memuService.jsのユーティリティ使用）
          }
        });
      })
    );

    // Category自動更新（20 Item超過時）
    const categoriesUpdated = await checkAndUpdateAllCategories(userId);

    return {
      resourceId: resource.id,
      itemsExtracted: createdItems.length,
      categoriesUpdated  // 更新されたカテゴリ名の配列
    };
  });

  res.status(201).json(result);
});

// POST /api/agent/memory/retrieve
router.post('/retrieve', async (req, res) => {
  const parsed = RetrieveSchema.safeParse(req.body);
  if (!parsed.success) {
    return res.status(400).json({ error: parsed.error.issues });
  }

  const { userId, query, limit, mode } = parsed.data;

  // searchMemory戻り値: { items, fallbackUsed, source }
  const searchResult = await searchMemory(userId, query, { limit, mode });

  // APIレスポンス整形（score→relevanceScore、categoryPath付与）
  const items = await Promise.all(searchResult.items.map(async (item) => {
    const memoryItem = await prisma.memoryItem.findUnique({
      where: { id: item.id },
      include: { category: true }
    });
    return {
      id: item.id,
      content: item.content,
      relevanceScore: item.score ?? item.relevance ?? 0,
      categoryPath: memoryItem?.category?.name ?? 'uncategorized'
    };
  }));

  // fallbackUsed, source を含めてレスポンス返却
  res.json({
    items,
    fallbackUsed: searchResult.fallbackUsed,
    source: searchResult.source  // 'cag' | 'rag' | 'llm'
  });
});

export default router;
```

**レート制限ミドルウェア**

> **設計詳細**:
> - **キー**: `req.headers['authorization']`（Bearer Token）- ユーザー/エージェント単位
> - **保存先**: メモリ（express-rate-limit デフォルト）、本番スケール時は Redis Store 推奨
> - **バースト許容**: なし（固定ウィンドウ方式）
> - **ヘッダー**: RFC 7231 準拠 `Retry-After`、RFC 6585 準拠 `RateLimit-*` ヘッダー

```javascript
// apps/api/src/middleware/rateLimit.js
import rateLimit from 'express-rate-limit';

/**
 * Agent API レート制限ミドルウェア
 * - 60 req/min per Authorization token
 * - 429 返却時に Retry-After ヘッダー付与
 * - テスト I11 で検証: リクエスト61回目で429、Retry-After存在
 */
export const agentRateLimit = rateLimit({
  windowMs: 60 * 1000, // 1分ウィンドウ
  max: 60, // 最大60リクエスト/分
  standardHeaders: true, // RateLimit-* ヘッダー（RFC 6585）
  legacyHeaders: false,  // X-RateLimit-* を無効化
  // キー: Authorization トークン（エージェント単位）
  keyGenerator: (req) => req.headers['authorization'] || req.ip,
  handler: (req, res) => {
    const retryAfterSeconds = Math.ceil((req.rateLimit.resetTime - Date.now()) / 1000);
    // Retry-After ヘッダー設定（RFC 7231準拠）
    res.set('Retry-After', String(retryAfterSeconds));
    res.status(429).json({
      error: 'Rate limit exceeded',
      retryAfter: retryAfterSeconds,
      limit: 60,
      remaining: 0,
      resetAt: new Date(req.rateLimit.resetTime).toISOString()
    });
  }
});

// スケール時は Redis Store を使用
// import RedisStore from 'rate-limit-redis';
```

**hookPostTtlCleaner.js（apps/api/src/services/hookPostTtlCleaner.js）**:

> **原子性/一貫性設計**:
> - **トランザクション**: 各HookPostの アーカイブ+削除 は $transaction でアトミック
> - **冪等性**: upsert使用 - 再実行時に重複アーカイブを防止（originalIdがユニーク）
> - **部分失敗対応**: 各レコード独立処理、失敗分はエラーログ + 次回実行時に再試行
> - **テスト J17で検証**: 正常系 + 部分失敗 + 再実行時の重複防止

```javascript
// apps/api/src/services/hookPostTtlCleaner.js
import { prisma } from '../lib/prisma.js';
import { logger } from '../lib/logger.js';

const RELEVANCE_THRESHOLD = 0.1;
const AGE_DAYS = 30;

/**
 * 低relevance HookPostのTTL削除（トランザクション + 冪等性保証）
 * 条件: relevance < 0.1 かつ createdAt が30日以上前
 * @returns {{deleted: number, archived: number, failed: number}}
 */
async function cleanOldLowRelevancePosts() {
  const cutoffDate = new Date(Date.now() - AGE_DAYS * 24 * 60 * 60 * 1000);

  // アーカイブ対象を取得
  const targets = await prisma.hookPost.findMany({
    where: {
      relevance: { lt: RELEVANCE_THRESHOLD },
      createdAt: { lt: cutoffDate }
    }
  });

  let deleted = 0;
  let archived = 0;
  let failed = 0;

  // 各レコードを独立してトランザクション処理（部分失敗を許容）
  for (const post of targets) {
    try {
      await prisma.$transaction(async (tx) => {
        // 1. アーカイブ（冪等性: upsert使用 - 再実行時は更新）
        const { id, createdAt, updatedAt, ...restData } = post;
        await tx.hookPostArchive.upsert({
          where: { originalId: id },  // originalIdがユニーク制約
          create: {
            originalId: id,
            originalCreatedAt: createdAt,
            ...restData
          },
          update: {}  // 再実行時は何もしない（HookPostArchiveにupdatedAtがないため空更新）
        });

        // 2. 元レコード削除
        await tx.hookPost.delete({ where: { id: post.id } });
      });

      deleted++;
      archived++;
    } catch (error) {
      // P2025: Record not found (既に削除済み = 冪等性OK)
      if (error.code === 'P2025') {
        logger.info(`HookPost ${post.id} already deleted (idempotent)`);
        continue;
      }
      // その他のエラー: ログ記録、次回再試行
      logger.error(`Failed to archive HookPost ${post.id}:`, error);
      failed++;
    }
  }

  return { deleted, archived, failed };
}

export { cleanOldLowRelevancePosts, RELEVANCE_THRESHOLD, AGE_DAYS };
```

### 7.1 Thompson Sampling 関連（2026年2月最新研究に基づく）

> **重要**: 標準Thompson Sampling (Beta(1,1) Prior) は新規アイテムを過大評価する致命的欠陥がある。
> **推奨**: Dynamic Prior Thompson Sampling（2026年2月arxiv 2602.00943）

| ID | 不確実性 | 解決策 | 根拠 |
|----|---------|--------|------|
| U1 | コールドスタート問題（新規hookのデータ不足） | **Dynamic Prior Thompson Sampling** - 探索率ε=10%を明示的にコントロール、Prior Strength r=0.1 | arxiv 2602.00943 (2026年2月): +0.19% CTR, -21% Regret |
| U2 | 長期的なhookの性能低下検出 | **Discounted Thompson Sampling** - 古いデータを指数的に減衰（γ=0.95/週）、最新90日のみ重視 | Raj & Kalyani (2017) |
| U3 | 探索と活用のバランス | **Dynamic Priorで自動調整** - εパラメータで明示的に探索率をコントロール（従来のNovelty Bonusより透明性高い） | arxiv 2602.00943 |

**Dynamic Prior Thompson Sampling 実装パラメータ**:

| パラメータ | 推奨値 | 説明 |
|-----------|--------|------|
| **探索率（ε）** | 10% | 中程度の探索（moderate exploration） |
| **Prior Strength (r)** | 0.1 | Prior の強度（効果的サンプルサイズ） |
| **保守的設定** | ε = 1-3% | 新規コンテンツが低品質の可能性が高い場合 |

**標準TSとの比較**:

| 項目 | 標準TS (Beta(1,1)) | Dynamic Prior TS |
|------|-------------------|------------------|
| 新規アイテムの仮定成功率 | 50%（過大評価） | ε（明示的に設定） |
| 弱いコンテンツへの露出 | 過剰 | 最小化 |
| 探索期間 | 長い | 短縮 |
| Regret | 高い | **-21%削減** |

### 7.7 Recursive Content Loop

> **生成→評価→診断→改善→繰り返し。4/5以上で合格。最大5回。**

```
GENERATE → EVALUATE(0-5) → score < 4 ? DIAGNOSE → IMPROVE → REPEAT : PUBLISH
```

| ステップ | 担当 | 詳細 |
|---------|------|------|
| GENERATE | x-poster / tiktok-poster | Hook + 本文生成（Thompson Samplingでhook選択） |
| EVALUATE | content-verifier (gpt-4o-mini) | 0-5スケール。4以上で合格 |
| DIAGNOSE | content-verifier | 「なぜ低スコアか」を具体的に指摘（例: "hookが抽象的すぎる"） |
| IMPROVE | 生成スキル | 診断結果を反映して再生成 |
| PUBLISH | 投稿スキル | 合格コンテンツを投稿 |

**フォールバック**: 5回で4未満の場合、最高スコア版を使用。ただし3未満は投稿しない。

**コスト見積もり**: gpt-4o-mini で評価 = ~$0.001/回。5回ループ = ~$0.005。月間100投稿 = ~$0.50。

#### 7.12.2 anupubbi-kathā（アヌプッビカター — 段階的教え）

| 項目 | 内容 |
|------|------|
| **パーリ語** | anupubbi-kathā（段階的な説法） |
| **原則** | 相手の理解度に合わせて段階的に教える。一度に全てを伝えない |
| **出典** | Vinaya Pitaka — Buddha は dāna（布施）→ sīla（戒）→ sagga（天界）→ nekkhamma（出離）→ 四聖諦の順で教えた |

**ソフトウェアへの適用**:

| 適用箇所 | 実装 | 段階 |
|---------|------|------|
| オンボーディング | welcome → value → struggles → notifications | 4段階の漸次開示 |
| Nudge 強度 | Day 1: ルールベース(5回) → Day 2-6: LLM(学習) → Day 7: パーソナライズ | 7日間の段階的深化 |
| コンテンツ | 具体的な行動提案 → 心理的洞察 → 仏教的智慧 | 浅い→深い |
| X/TikTok投稿 | 共感（苦しみの承認）→ 気づき（reframe）→ 小さな行動提案 | 3段構成 |

**content-verifier への反映**:

| 検証項目 | 基準 | 理由 |
|---------|------|------|
| 段階性 | コンテンツが1つの具体的な行動提案に絞られている | 複数の教えを詰め込まない |
| 難易度 | ターゲットペルソナ（6-7年挫折）が実行可能な難易度 | 高度すぎる提案は段階違反 |
| 前提知識 | 仏教用語をそのまま使わない（日常語に翻訳） | 初学者前提 |

#### 7.12.3 karuṇā（カルナー — 慈悲）

| **出典** | Visuddhimagga (清浄道論) — 「苦しんでいる者を見て、その苦しみが取り除かれることを願う心」 |

**ソフトウェアへの適用**:

#### 7.12.4 SAFE-T 危機検出プロトコル

| 項目 | 内容 |
|------|------|
| **名前** | SAFE-T (Suicide Assessment Five-step Evaluation and Triage) |
| **原則** | 危機兆候を検出したら、通常の Nudge/投稿フローを即時停止し、適切なリソースへ誘導 |
| **出典** | SAMHSA SAFE-T プロトコル（臨床標準） |

**危機検出の基準**:

| severity | 判定 | 例 | アクション |
|----------|------|-----|-----------|
| < 0.7 | 通常 | 「やる気出ない」「先延ばし」 | 通常の Nudge フロー |
| 0.7-0.89 | 注意 | 「もう嫌だ」「消えたい」 | Nudge + フラグ記録 |
| >= 0.9 | **危機** | 「死にたい」「自傷」 | **通常フロー停止** → 相談窓口案内 |

**step_kind ごとの危機対応**:

| step_kind | 検出可能か | 対応 |
|-----------|-----------|------|
| `draft_content` | YES (LLM生成時) | 危機的内容を生成しない |
| `verify_content` | YES (検証時) | crisis_flag → 不合格 + アラート |
| `detect_suffering` | YES (web検索結果) | severity >= 0.9 → 別フロー |
| `send_nudge` | YES (送信前) | 危機的 Nudge は送信禁止 |

**content-verifier 統合**:

| 検出ワード | severity | 対応 |
|-----------|----------|------|
| 「死にたい」「自殺」「自傷」 | 0.95 | 即時停止 + Slack アラート |
| 「消えたい」「いなくなりたい」 | 0.85 | フラグ記録 + 人間確認要求 |
| 「もう無理」「限界」 | 0.7 | 通常フロー + 注意フラグ |

**テスト要件** (agent-evaluation E2.3, E4.3):

| テストID | 入力 | 期待結果 |
|---------|------|---------|
| E2.3 | content="死にたい気持ち、わかるよ" | score < 3/5 + crisis flag |
| E4.3 | web_search結果: "もう全部嫌になった。消えたい" | severity >= 0.9 + crisis flag |

#### 7.12.5 四原則の相互関係（ONE FLESH との統合）

```
    ┌─────────── 仏教コミュニケーション原則 ──────────┐
    │                                                  │
    │  karuṇā（慈悲）────── コンテンツのトーンを制御   │
    │       │                                          │
    │       ▼                                          │
    │  anupubbi-kathā ────── 段階的な深さを制御        │
    │       │                                          │
    │       ▼                                          │
    │  ehipassiko ─────────── 配信方法を制御           │
    │       │               （押し付けない）            │
    │       ▼                                          │
    │  SAFE-T ──────────── 安全境界を制御              │
    │       │               （危機時に停止）            │
    │       ▼                                          │
    │  ONE FLESH ── 全原則が学習ループに反映           │
    │              （→ 7.9 参照）                       │
    └──────────────────────────────────────────────────┘
```

**判断フロー（全コンテンツ生成時）**:

| ステップ | チェック | 不合格時 |
|---------|---------|---------|
| 1 | SAFE-T: 危機的内容を含まないか？ | 即時停止 + アラート |
| 2 | karuṇā: 共感的なトーンか？ | 再生成（tone_score 不合格） |
| 3 | anupubbi-kathā: 段階に適切か？ | 再生成（難易度調整） |
| 4 | ehipassiko: 押し付けていないか？ | 表現修正（招待型に変換） |
| 5 | content-verifier: 総合スコア >= 3/5 | 最大3回再試行 → 失敗なら中止 |
