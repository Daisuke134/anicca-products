**Trial period テキスト生成（動的）:**

```swift
private var trialPeriodText: String? {
    guard let discount = selectedPackage?.storeProduct.introductoryDiscount else { return nil }
    let period = discount.subscriptionPeriod
    let isJA = Locale.current.languageCode == "ja"
    switch period.unit {
    case .day:   return isJA ? "\(period.value)日間" : period.value == 1 ? "1 Day" : "\(period.value) Days"
    case .week:  return isJA ? "\(period.value)週間" : period.value == 1 ? "Week" : "\(period.value) Weeks"
    case .month: return isJA ? "\(period.value)ヶ月" : period.value == 1 ? "1 Month" : "\(period.value) Months"
    case .year:  return isJA ? "\(period.value)年" : period.value == 1 ? "1 Year" : "\(period.value) Years"
    @unknown default: return nil
    }
}
```

**PostHog イベント tracking（Mixpanel と並行 — Mixpanel は削除しない）:**

| PostHog イベント | いつ | properties | 対応 Mixpanel イベント |
|----------------|------|-----------|---------------------|
| `paywall_viewed` | paywall 表示時 | `variant` | `paywall_plan_selection_viewed`（variant なし） |
| `paywall_purchased` | RevenueCat 購入成功時 | `variant`, `plan_type`, `has_trial` | `onboarding_paywall_purchased`（variant なし） |
| `paywall_dismissed` | ✕ or Maybe later タップ時 | `variant` | `onboarding_paywall_dismissed_free`（variant なし） |

**Mixpanel は残す。** 既存の AppMetrics cron / OpenClaw スキルが Mixpanel から読んでる。PostHog は A/B テスト専用レイヤーとして追加。

**自動改善サイクル:**

```
Week 1: Variant A vs B → PostHog が統計的有意差を検出
  → Winner をベースに
  → テキスト1要素だけ変えた新 Variant を作る
  → Winner vs 新 Variant で新 Experiment
  → 繰り返し
  → paywall が自己改善（Screenshot PPO と同じ原理）
```

## 6. 不確実な点（実装前に解決）

| # | 点 | 解決方法 |
|---|---|---------|
| 1 | PostHog SDK `sessionReplay` API 名がv3で正しいか | ビルド時に確認。エラーなら SDK ヘッダーで正しいプロパティ名を確認 |
| 2 | `getFeatureFlag()` 戻り値が `String` cast 可能か | PostHog docs 標準パターン。nil 時は `"control"` にフォールバック |
| 3 | feature flag がオンボーディング到達時にロード済みか | `PostHogSDK.shared.reloadFeatureFlags()` を safety net として入れる |
| 4 | `Purchases.shared.appUserID` が AppDelegate 時点で利用可能か | `SubscriptionManager.configure()` が先に呼ばれるので OK |
| 5 | pbxproj 手動編集で Xcode が壊れないか | ビルドで即確認。既存パッケージと同じパターン |
