# Anicca iOS 1.6.3 Spec — Viral Nudge Evaluation Loop (TikTok/X/iOS)

最終更新: 2026-02-12 JST  
対象: 1.6.3（実装用・Codex Review用）

---

## 0. 背景と目的

### 背景
1.6.2運用で以下の課題が顕在化した。
- TikTok投稿で「画像は出せるが、画像自体のフック品質が担保されない」
- 文字化け・可読性不足など、投稿前品質ゲート不在による事故
- “生成できる”と“投稿して成果を出せる”の間に、評価ループが不足

### 目的
「No Verify, No Post」を強制し、
- **投稿前品質担保**
- **不合格時の再生成ループ**
- **投稿後学習ループ**
を閉ループ化する。

特にTikTokは「画像そのものがフック」である前提で最適化する。

---

## 1. スコープ

### In Scope（1.6.3）
1. TikTok投稿前の画像評価ゲート（必須）
2. 評価スコア閾値（>=85）での投稿可否判定
3. 不合格時の自動再生成（最大10回）
4. TikTok投稿時 `autoAddMusic: true` のデフォルト化
5. 投稿後の `postSubmissionId/status/publicUrl` 取得と保存
6. 同型ルールのX/iOSへの展開（最小実装）

### Out of Scope（次フェーズ）
- マルチクリエイティブA/B自動最適化（多腕バンディット本格導入）
- TikTok広告配信（Ads API連携）
- IG/Reels/Reddit等の同時最適化

---

## 2. 設計原則（固定）

1. **No Verify, No Post**  
   検証通過前の投稿禁止。

2. **No Threshold, No Ship**  
   スコア閾値未達は投稿禁止。

3. **3 Strikes Stop**  
   最大10回再生成して未達なら fallback に切替。

4. **Evidence First**  
   実行証跡（submissionId/publicUrl/status/error）を必ず記録。

5. **Single Source of Truth**  
   判定・実行・証跡はops/event/auditストアに一元化。

---

## 3. TikTok 投稿フロー（確定仕様）

## 3.1 生成
入力:
- suffering topic（例: rumination / self-hatred / burnout）
- target persona（例: 仕事で叱責されて落ち込む人）
- slot（morning/evening）

出力:
- 1080x1920 画像1枚（まずは単発）
- caption

画像内必須要件:
- 共感 1行
- 具体行動 1行
- 許可/安心 1行
- 合計3行以内

## 3.2 投稿前評価（TikTok Image Gate）
評価項目（100点）:
- Hook停止力: 30
- 可読性: 25
- 共感深度: 20
- 行動具体性: 15
- ブランド整合: 10

合格条件:
- 合計 >= 85
- かつ以下Hard Failが0件

Hard Fail条件:
- 文字化け
- 主要テキスト判読不能
- 断罪・煽り・医療断定
- ブランド違反（Aniccaトーン逸脱）

## 3.3 再生成ループ
- 不合格 → prompt改善して再生成
- 最大10回
- 3回不合格 → 投稿中止 + failure reason保存

## 3.4 TikTok投稿（Blotato）
投稿payload必須:
- `targetType: "tiktok"`
- `privacyLevel: "PUBLIC_TO_EVERYONE"`
- `disabledComments: false`
- `disabledDuet: false`
- `disabledStitch: false`
- `isBrandedContent: false`
- `isYourBrand: false`
- `isAiGenerated: true`
- `autoAddMusic: true`  ← 1.6.3でデフォルト

備考:
- TikTokはmedia必須（image/videoなしは422）

## 3.5 公開確認
- publish response から `postSubmissionId` 取得
- `GET /v2/posts/{postSubmissionId}` をポーリング
- `status=published` と `publicUrl` を保存
- timeout/failureはエラー理由保存

## 4. X / iOS への同型展開（1.6.3）

## 4.1 X

## 8. リスクと対策

1. **過剰厳格で投稿数低下**
   - 対策: slotごとに最低投稿数ガード（fail時はfallbackテンプレ）

2. **評価器の過学習**
   - 対策: 週次で評価基準を実績に合わせ再調整

3. **TikTok API仕様変更**
   - 対策: publish失敗コードを分類し、即時アラート

## 11. Patch Snippets（実装断片）

### 11.1 TikTok publish payload（music default）
```ts
const payload = {
  post: {
    accountId: process.env.BLOTATO_TIKTOK_ACCOUNT_ID,
    content: {
      text: caption,
      mediaUrls: [imageUrl],
      platform: "tiktok",
    },
    target: {
      targetType: "tiktok",
      privacyLevel: "PUBLIC_TO_EVERYONE",
      disabledComments: false,
      disabledDuet: false,
      disabledStitch: false,
      isBrandedContent: false,
      isYourBrand: false,
      isAiGenerated: true,
      autoAddMusic: true,
    },
  },
};
```

### 11.2 Verify loop（10回 + fallback）
```ts
let best: { score: number; creative: Creative } | null = null;

for (let attempt = 1; attempt <= 10; attempt++) {
  const creative = await generateCreative(context, attempt);
  const verdict = await evaluateCreative(creative);

  if (!best || verdict.score > best.score) best = { score: verdict.score, creative };
  await saveEvalAttempt({ attempt, score: verdict.score, pass: verdict.pass, reasons: verdict.reasons });

  if (verdict.pass && verdict.score >= 85) {
    return { creative, mode: "normal" };
  }

  context = await refinePromptContext(context, verdict.reasons);
}

return { creative: buildSafeFallbackCreative(context, best?.creative), mode: "fallback" };
```

### 11.3 Post status polling（ID/URL証跡）
```ts
const submit = await blotatoPublish(payload);
const id = submit.postSubmissionId;

for (let i = 0; i < 10; i++) {
  const s = await blotatoGetPost(id);
  if (s.status === "published") {
    await saveEvidence({ submissionId: id, status: s.status, publicUrl: s.publicUrl });
    return s;
  }
  if (s.status === "failed") {
    await saveEvidence({ submissionId: id, status: s.status, error: s.error });
    throw new Error(s.error || "publish failed");
  }
  await sleep(3000);
}

await saveEvidence({ submissionId: id, status: "timeout" });
throw new Error("publish timeout");
```
