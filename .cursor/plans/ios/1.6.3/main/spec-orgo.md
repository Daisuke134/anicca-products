## 3. Orgo とは何か

| 項目 | 内容 |
|------|------|
| 一言 | **Desktop infrastructure for AI agents**。AI が「画面を見て・マウス/キーボードで操作する」ためのクラウド VM を用意するサービス。 |
| Computer use | AI が画面をキャプチャし、クリック・入力・スクロールなどを実行する技術。Anthropic の Computer Use などが該当。Orgo はその実行環境をクラウドで提供する。 |
| 提供するもの | ヘッドレスなクラウド VM（500ms 以内にブート）。1 VM = 1「コンピュータ」。その VM 上で AI がスクリーンショット取得・クリック・タイプ・コード実行を行う。 |
| エージェントループ | See（画面取得）→ Decide（AI が判断）→ Act（操作）→ 繰り返し。 |

## 4. 他ツールとの違い（Unbrowse / P0 等）

| 観点 | Orgo | Unbrowse（OpenClaw プラグイン） | P0（元 Twitter CEO 関連の並列実行など） |
|------|------|---------------------------------|----------------------------------------|
| 役割 | **「AI が使う 1 台の PC」をクラウドで貸す**。画面あり・マウス/キーボード操作あり。 | **スキル生成・API 逆引き**。サイトをブラウズしてネットワークをキャプチャし、スキル（SKILL.md + クライアント）を自動生成。PC を貸すわけではない。 | 文脈により **並列エージェント・並列タスク実行** のイメージ。複数サブエージェントを同時に走らせる仕組み。 |
| 提供単位 | 1 API 呼び出し = 1 Computer（VM）。複数立てれば「サブエージェントそれぞれに 1 台ずつ PC」。 | 1 プラグイン = スキル発見・生成。OpenClaw のツールとして「新しいスキルを自動で作る」。 | （P0 は製品名・仕様が文脈依存のため）並列化そのもの。Orgo と組み合わせると「各サブエージェントに Orgo の Computer を 1 台ずつ割り当てる」構成が可能。 |
| Anicca での位置づけ | 「画面操作が必要なタスク」を Anicca にやらせたいときに、その実行環境として使う。 | 既に VPS で導入済み。スキルの自動生成・逆引き用。 | 並列提案・並列 mission の実行基盤。Orgo は「その各実行先に PC を 1 台ずつ」与えるオプション。 |

要約: **Unbrowse = スキルを増やす道具。Orgo = エージェントに「1 台の PC」を割り当てる道具。P0 的並列 = 複数エージェントを同時に走らせる考え方で、Orgo はその各エージェントに VM を割り当てる先。**

## 5. Anicca で Orgo を入れたときに起こること・体験

| 項目 | 内容 |
|------|------|
| 技術的に起こること | OpenClaw（Anicca）が Orgo API を叩き、必要なときに「Computer」を 1 台以上起動。各 Computer 上で AI がブラウザやデスクトップを操作する。サブエージェントを複数立ち上げる場合、それぞれに Orgo の Computer を 1 台ずつ持たせられる。 |
| ユーザー体験 | ユーザーは Slack で「これ調べて」「この手続きの画面とって」などと依頼する。Anicca が裏で Orgo の VM を起動し、ブラウザで検索・フォーム入力・スクショ取得などを実行し、結果を Slack に返す。 |
| 苦しみの軽減でできること | 「自分でブラウザを開いて同じことを何度もやる」負荷を減らす。例: 申込フォームの下書き入力、複数サイトの情報収集、スクショ付きレポートの自動作成。習慣化が苦手なユーザー向けには「手続きの下準備を Anicca がやっておく」ことで、実際に人がやるステップを減らす。 |

---

## 6. Anicca が「苦しみを減らす」文脈で Orgo でできそうなこと

| 用途 | 具体例 |
|------|--------|
| 情報の取りまとめ | 「このテーマで今のトレンドをまとめて」→ Orgo 上でブラウザを開き検索・スクショ → 要約と画像を Slack に返す。 |
| 手続きの下準備 | 「〇〇の申し込みに必要な情報をフォーム用に整理して」→ Orgo で該当サイトを開き、入力項目を洗い出し、ユーザーが書きやすい形で返す（または Human in the Loop で確認後に送信）。 |
| 定期的な確認の代行 | 「毎週このページの更新をチェックして」→ Orgo で該当 URL を開き、差分や要約を報告。ユーザーは「見るだけ」に集中できる。 |
| 複数タスクの並列 | 提案・mission で「A と B と C をそれぞれ調べて」と分解し、サブエージェントそれぞれに Orgo の Computer を 1 台ずつ割り当て、並列でブラウザ操作。結果を 1 つにまとめて返す。 |

いずれも「人間がブラウザで何度も同じ操作をする負荷」を減らし、判断や最終確認は人間が行う前提にできる。

## 7. 実装する場合のメモ

| 項目 | 内容 |
|------|------|
| 前提 | Orgo API キーを取得し、`ORGO_API_KEY` で .env に保持。 |
| OpenClaw 側 | Orgo API を呼ぶツール or スキルを 1 本用意。Computer の create / prompt / screenshot / 破棄 などをラップする。 |
| サブエージェント × Orgo | 並列サブエージェントを立てる既存/今後の仕組みに、「各エージェントに Orgo Computer を 1 台割り当てる」オプションを足す。 |
| コスト・制限 | Orgo の従量（VM 時間・リクエスト）に依存。長時間・大量並列はコストとレート制限を要確認。 |
| 倫理・安全 | フォーム送信・決済・個人情報入力は Human in the Loop 必須。Anicca のセキュリティ・倫理ルール（spec-openclaw-learnings 等）に合わせる。 |
