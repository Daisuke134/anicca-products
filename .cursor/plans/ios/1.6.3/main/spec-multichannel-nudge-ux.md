# 1.6.3 マルチチャネル Nudge UX 実装スペック（確定版）

作成: 2026-02-11  
対象: Anicca 1.6.3  
目的: 「全チャネル同列の介入ツール」という前提で、UX中心に実装を固定する。

---

## 0. この仕様の前提（固定）

1. iOS / X / TikTok / Newsletter / Moltbook はすべて同列の Nudge 実行チャネル。  
2. 送客前提（どこかへ誘導する設計）は採用しない。各チャネル内で介入を完結させる。  
3. 個人のクロスチャネル突合はしない（例: X アカウントと Newsletter 購読者IDを紐づけない）。  
4. Moltbook は AI エージェント専用SNSとして扱う。  
5. 目的指標は「苦しみ低減」。エンゲージメントは補助指標。
6. Anicca は「プロアクティブ仏教エージェント」として、開発・運用・マーケティングを自律的に回す（human approval は高リスク操作のみ）。

## 0.1 自律運用の定義（1.6.3）

| 項目 | 定義 |
|---|---|
| 自律開発 | 仕様作成、実装、テスト、デプロイ準備、回帰検知、改善提案を定期ループで実行する |
| 自律マーケ | X/TikTok/Newsletter/iOS向け介入コンテンツ生成、配信計画、効果計測、改善を定期ループで実行する |
| 人間の役割 | 方向性決定、境界設定、ハイリスク承認、最終監督 |
| 非許容 | 無制限投稿、無承認デプロイ、無承認課金・返金、安全ルール無視 |

---

## 1. UXゴール（ユーザーに何が起きるべきか）

### 1.1 体験ゴール

| ゴール | 定義 |
|---|---|
| すぐ効く | 1接触あたり 10〜90秒で実行可能な行動が1つ提示される |
| 責めない | 文体は非難・説教を避け、自己受容と小さな実行に寄せる |
| 続いている感じ | 「毎回ゼロから」ではなく、前回反応を踏まえた次の一歩が出る |
| その場完結 | 投稿/通知/記事単体で介入が完了する |

### 1.2 非ゴール

| 非ゴール | 理由 |
|---|---|
| チャネル横断の個人追跡最適化 | 現在の運用要件に反する |
| アプリ送客最適化 | 本仕様の目的は送客ではなく介入完結 |
| 常時チャット化 | 認知負荷と依存リスクを上げる |

## 2. チャネル別 UX 仕様（具体）

## 2.1 iOS（通知 + Nudge Card）

### ユーザー体験
- 通知で「最小行動1つ」を提示する。  
- 開封後のカードで `やる / あとで` など 1〜2択。  
- 終了時に「次回の一歩」を1文表示して閉じる。  

### 表示文フォーマット（固定）
1. 共感 1文  
2. 再解釈 1文  
3. 行動 1つ（10〜90秒）

### 例
- 「いま重いよね。壊れてるんじゃなくて過負荷。10秒だけ肩を下げよう。」

---

## 2.2 X（投稿単体で介入）

### ユーザー体験
- 1投稿を読んだ時点で介入が成立する。  
- CTAは外部遷移ではなく、その場でできる行動（3秒〜30秒）を明示する。  

### 例
- 「頑張れない日は故障じゃない。回復中。いま3秒だけ肩の力を抜く。」

---

## 2.3 TikTok（15〜30秒動画介入）

### ユーザー体験
- 動画内で `共感 → 再解釈 → 実行` を完結させる。  
- 視聴中に実行できる最小行動を必ず含める。  

### 例構成
1. 「集中が切れてしんどいよね」  
2. 「意志不足じゃなく神経の過負荷」  
3. 「今10秒だけ遠くを見る」

---

## 2.4 Newsletter（週次の構造化介入）

### ユーザー体験
- 週次で「実行可能な3介入」を固定フォーマットで提供。  
- 読了時に「今週はこれだけやる」が明確になる。  

### 例
1. 朝10秒: 姿勢を整える  
2. 昼90秒: 画面から離れる  
3. 夜15秒: 今日の最小達成を確認する

---

## 2.5 Moltbook（AIエージェント間の介入文法実験）

### 役割
- 人間送客導線ではなく、介入文法の実験・評価チャネル。  
- 反応の良い短文パターンを抽出し、他チャネル文面改善に再利用する。  

### 制約
- AI専用SNSとして運用。  
- 危機文脈では SAFE-T ルールを最優先する。

---

## 3. 「生きてる感じ」の実装定義

「生きてる感じ」は以下3条件を満たすこと。

1. 前回反応を覚える（チャネル内のみ）  
2. 次の一歩を1段だけ出す  
3. 同じ介入意図を媒体形式に合わせて変換する

補足: 「同一人物を全チャネルで追跡する」ことではない。

---

## 4. 実装仕様（完全対応）

## 4.1 共通介入フォーマット

### 仕様
- すべての生成物に `empathy`, `reframe`, `micro_action` を持たせる。  
- `micro_action` は 1つのみ。  
- 文字数上限はチャネル別テンプレで制約する。

### データ項目（内部）
- `previous_outcome`（例: opened / ignored / thumbs_up / thumbs_down）  
- `next_micro_step`（次に出す1行動）  
- `tone_variant`（gentle / direct など）

---

## 4.2 API（mobile nudge）

対象: `apps/api/src/routes/mobile/nudge.js`

実装:
1. `/pending` 応答に `continuity_line` を追加  
2. `/today` 応答に `continuity_line` を追加  
3. `/feedback` 受信時に `previous_outcome` と `next_micro_step` を更新可能にする

受け入れ:
- iOS側で前回反応に応じた文面差分が表示される。

---

## 4.3 iOS

対象:
- `aniccaios/aniccaios/Views/NudgeCardView.swift`  
- `aniccaios/aniccaios/Services/ServerNudgeInboxService.swift`  
- `aniccaios/aniccaios/Services/NudgeTriggerService.swift`

実装:
1. NudgeCard に `continuity_line` 表示欄を追加  
2. 主アクション後に「1回だけ追問」（2ボタン固定、自由入力なし）  
3. 追問結果を既存フィードバック送信に追加

受け入れ:
- 1セッション内で最大1回の追問に制限される。  
- UX完了時間は 90秒以内を維持する。

---

## 4.4 X / TikTok / Newsletter 生成運用

対象: OpenClaw スキル運用とテンプレ群

実装:
1. Xテンプレ: 1投稿完結型（共感1文+再解釈1文+行動1つ）  
2. TikTok台本: 15〜30秒の3段構成固定  
3. Newsletterテンプレ: 週3介入の固定枠

受け入れ:
- 各コンテンツに実行可能アクションが1つ含まれる。  
- 遷移前提の文言を含めない。

---

## 4.5 Moltbook（AI-only）

対象: Moltbook監視/投稿の介入文法評価フロー

実装:
1. 反応が良かった短文を `hook_candidates` へ反映  
2. 危機文脈の文法を blacklist 化（安全側に倒す）  
3. AI間の議論出力は「介入文法の改善」に限定

受け入れ:
- Moltbook由来の改善が他チャネル文面に再利用される。  
- SAFE-T違反ゼロ。

---

## 5. マルチエージェント運用（内部）

役割分離:
1. 観測役: 苦しみシグナル要約  
2. 介入役: 候補生成  
3. 安全役: 危機遮断  
4. 媒体役: 形式変換

heartbeat:
- 定期 check-in で短い議論を発生させ、毎回「採用1案」に収束させる。

注意:
- ユーザー向けは単一の明確な介入として出す（内部の複雑性を露出しない）。

## 5.1 自律開発・自律マーケ運用ループ（追加）

毎日ループ（自動）:
1. 前日KPI集計（苦しみ低減 + 収益）  
2. 改善仮説を3件生成  
3. 小変更を実装（文面、テンプレ、配信スロット、軽微コード）  
4. テスト/検証  
5. 安全チェック通過後に反映

毎週ループ（自動+承認）:
1. 週次レトロスペクティブ生成  
2. 仕様更新案（spec差分）生成  
3. 高影響変更（大規模配信変更、本番デプロイ、課金変更）は承認待ち  
4. 承認後に適用

---

## 5.2 自律実行の権限境界（必須）

自動許可（default allow）:
- コンテンツ下書き、要約、分析、A/B案生成  
- read-only データ取得  
- 低リスク設定変更（文面・テンプレ・スケジュールの範囲内）

承認必須（human gate）:
- 本番デプロイ  
- 外部への公開投稿の大量実行（閾値超え）  
- 課金/返金/契約に関わる操作  
- データ削除、アカウント権限変更  
- SAFE-Tルール変更

---

## 6. 計測（個人突合なし）

## 6.1 必須KPI（チャネル内）

| チャネル | 必須KPI |
|---|---|
| iOS | 開封率、行動実行率、👍率、無反応率 |
| X | 反応率（媒体内）と投稿完結率（行動文が入っている比率） |
| TikTok | 3秒維持率、完視聴率、行動文到達率 |
| Newsletter | 開封率、介入セクション到達率 |
| Moltbook | 文法改善採用率、安全逸脱率 |

## 6.2 ガードレール
- 危機時は通常介入を中断（SAFE-T優先）  
- 説教・断定・羞恥喚起の文法を禁止  
- 1コンテンツ1アクションを維持

---

## 7. 受け入れ基準（DoD）

1. 全チャネルで「その場完結の介入」が成立している。  
2. 全チャネルで「次の一歩は1つのみ」が守られている。  
3. iOSで前回反応ベースの `continuity_line` が表示される。  
4. 個人クロスチャネル突合が実装されていない。  
5. SAFE-Tルールを破る出力がない。  
6. MoltbookがAI-only運用前提として明記されている。

---

## 8. 実装順（1.6.3）

1. API: `continuity_line` と feedback拡張  
2. iOS: continuity表示 + 追問1回  
3. テンプレ更新: X/TikTok/Newsletter  
4. Moltbook文法評価反映  
5. KPI計測と週次改善
6. 自律開発/自律マーケの日次・週次ループをcron化  
7. 承認ゲート付き実行ポリシーを適用
