## 背景

1.8.0 (Build 7) が Apple Review で Reject。以下の修正 + TikTok SDK 強化を 1.8.1 として再提出。

## Rejection Issues

### Issue 1: Guideline 2.5.1 — HealthKit not identified in UI

**Apple の指摘:** HealthKit API を使用しているが、UI に HealthKit 機能が明示されていない。

**根本原因:** entitlements に `com.apple.developer.healthkit = true` + Info.plist に usage descriptions があるが、**Swift コードに HealthKit の import が 0 行**。実際に使っていない機能の entitlement が残っている。

**修正方針:** HealthKit を実際に使っていないので、entitlement と usage descriptions を完全削除。

### Issue 2: Guideline 1.4.1 — Medical Disclaimer missing

**Apple の指摘:** 医療診断や治療アドバイスを提供しているが、必要な医療免責事項がない。

**修正方針:** App Store description の末尾に医療免責事項を追記（全6ロケール）。

```diff
+    /// SKAdNetwork conversion value 更新
+    /// iOS 16.1+: fine + coarse value, iOS 15.4+: fine value のみ
+    /// Source: https://developer.apple.com/documentation/storekit/skadnetwork/updatepostbackconversionvalue(_:coarsevalue:completionhandler:)
+    func updateSKANConversionValue(_ value: Int) {
+        if #available(iOS 16.1, *) {
+            let coarse: SKAdNetwork.CoarseConversionValue = value >= 3 ? .high : value >= 1 ? .medium : .low
+            SKAdNetwork.updatePostbackConversionValue(value, coarseValue: coarse) { error in
+                if let error { print("[SKAN] update failed: \(error)") }
+            }
+        } else if #available(iOS 15.4, *) {
+            SKAdNetwork.updatePostbackConversionValue(value) { error in
+                if let error { print("[SKAN] update failed: \(error)") }
+            }
+        }
+        logger.debug("SKAN conversion value updated: \(value)")
+    }
```
