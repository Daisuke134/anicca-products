# OpenClaw / Anicca: VPS 絶対ルール・絶対禁止事項

**Anicca の実行環境は VPS。正しい状態は VPS にしかない。** このファイルは全エージェントが必ず守る絶対事項である。

---

## 絶対ルール（やること）

| # | ルール | 内容 |
|---|--------|------|
| 1 | **VPS への反映は必須** | 仕様・fix・SKILL に「VPS に反映する」「VPS で確認する」と書いてあれば、**エージェントが自分で SSH し、VPS 上で編集・確認・反映まで完了する。**「VPS に反映してください」とユーザーに言って終わりにしない。 |
| 2 | **SSH はエージェントが実行する** | `ssh anicca@46.225.70.241` で接続できる。必要な編集・確認・再起動は **エージェントがターミナルで SSH を実行して行う。** |
| 3 | **ローカル編集は「反映の前準備」に過ぎない** | ローカルでファイルを編集するのは、**VPS に送る内容を用意するため**だけ。編集しただけでは 1mm も本番に効いていない。**必ず scp / rsync / VPS 上での編集のいずれかで VPS に反映する。** |

---

## 絶対禁止事項（やってはいけないこと）

| # | 禁止事項 | 説明 |
|---|----------|------|
| 1 | **「VPS にアクセスできません」「SSH できません」と言い切ること** | **禁止。** 上記 SSH で接続できる。できないと言い訳しない。 |
| 2 | **ユーザーにコマンドを叩かせない** | 「VPS に SSH してこのコマンドを実行してください」「ターミナルで scp を実行してください」は **禁止。** 全ての情報はエージェントが持っている。エージェントが自分で実行する。 |
| 3 | **ローカルだけで確認して「VPS も同じはず」とすること** | ローカルでファイルを確認し、VPS の状態を確認せずに「ローカルではこうなっているので VPS もこうなっています」と断言するのは **禁止。** 確認は VPS で行う。 |
| 4 | **ローカルで編集して「直りました」で終わること** | ローカルでファイルを編集しただけで「修正しました」と報告するのは **禁止。** VPS に反映して初めて直ったことになる。反映していなければ未完了である。 |
| 5 | **ユーザーへの有罪推定的な質問** | 「あなたが VPS に反映しましたか？」など、やるべきことをユーザーに押し付ける質問は **禁止。** やるべきはエージェントである。 |
| 6 | **上記を破る言い訳** | 「SSH しない方針なので」「VPS にはアクセスできない前提で」など、禁止事項を正当化する言い訳は **禁止。** |
| 7 | **ファイル全体の上書き** | **絶対禁止。** VPS に反映するとき、jobs.json や SKILL を「ファイル全体を別内容で置き換える」ようにしてはいけない。あなたが把握していないスキル・cron が VPS にあり、上書きで消えている可能性がある。**直すべき箇所を確認し、その箇所だけを直す（差分・部分更新）。** ファイルごと差し替えるような運用は禁止。 |
