# Visual QA 採点プロンプト

## 使い方

`workspace/export/` の PNG を vision model に送信し、以下の 5カテゴリで採点する。
EN 4枚 + JA 4枚を**それぞれ別セット**として採点。

## 採点プロンプト

```
以下の App Store スクリーンショット 4枚を採点してください。
各カテゴリ 10点満点、合計 50点満点。

## Critical Checks（1項目でも FAIL なら即 FAIL）
- 全4枚のフォントサイズが統一されているか（±10px以内）
- iPhone フレームの背景が浮いていないか（白いboxが見えてはいけない）
- ヘッドラインとサブテキストの両方が全枚に存在するか
- テキストがデバイス画面と重なって読めない箇所がないか

## カテゴリ

1. Clarity (10pt)
   - ヘッドラインが App Store サムネイルサイズ（小さい）で 1秒以内に読めるか
   - フォントサイズは十分大きいか
   - テキストと背景のコントラスト比は十分か

2. Hierarchy (10pt)
   - 視線の流れ: ヘッドライン → デバイス画面 → 背景
   - 最も重要な情報が最も目立つか
   - デバイスフレームがアプリ画面を適切にフレーミングしているか

3. Consistency (10pt)
   - 4枚のカラーパレット統一
   - フォント・サイズ・ウェイトの統一
   - レイアウトバリエーション（隣接スライドが同じ配置でないか）

4. Conversion (10pt)
   - 「ダウンロード」したくなるか
   - Screen 1 が最強の訴求になっているか
   - 各スライドが独立して 1つの価値を伝えているか

5. Technical (10pt)
   - 解像度は 1284x2778 か
   - 切り抜き / クロップに問題ないか
   - デバイスフレームのアライメントは正確か

## 出力フォーマット

Critical: PASS / FAIL（理由）

| カテゴリ | スコア | コメント |
|---------|--------|---------|
| Clarity | X/10 | ... |
| Hierarchy | X/10 | ... |
| Consistency | X/10 | ... |
| Conversion | X/10 | ... |
| Technical | X/10 | ... |
| **合計** | **XX/50** | |

## 判定: PASS (40+) / FAIL (39以下)
改善点: [リスト]
```

## 判定テーブル

| 結果 | アクション |
|------|-----------|
| Critical FAIL | → 即 FAIL。PHASE 4 に戻る（技術的修正） |
| 40/50+ | → PHASE 6 へ |
| 39/50 以下 | → PHASE 3 に戻る（ヘッドライン再生成） |
| 3回連続 FAIL | → Slack 警告 → EXIT |
