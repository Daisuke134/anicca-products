# ENOSPC 対策メモ（2026-02-14）

## 原因は「2種類」あった

1. **メインディスク（Data）**  
   DerivedData 約59GB → 削除して約28GB確保。現在 **約71GB 空き**。
2. **シミュレータ用ボリューム**  
   iOS Simulator の各ランタイム用ボリュームが **約98%使用**（空き 400MB 前後）。  
   ここに書き込む処理（シミュレータ起動・アプリのログ/キャッシュなど）で ENOSPC が出る。

## 実施した対応

| 対応 | 効果 |
|------|------|
| DerivedData 削除 | Data 約60GB 空き |
| シミュレータ「利用不可」デバイス削除 | 軽微 |
| **iOS 16.4 ランタイム削除** | そのランタイム用ボリュームを解放 |
| **重複 iOS 26.1（23B80）ランタイム削除** | 約16GB 解放（Deleting 完了後に反映） |

## まだ ENOSPC が出る場合

**どの操作でエラーになるか**を特定する必要がある。

- **Node / npm / OpenClaw / 一般アプリ**  
  → 通常は Data に書くので、現状の空きなら ENOSPC は出にくい。  
  それでも出る場合は、そのコマンド・アプリ名と **エラーメッセージ全文** を控える。
- **Xcode でシミュレータ実行・ビルド**  
  → シミュレータのランタイムボリューム（約400MB しか空きがない）に書いている可能性が高い。  
  → 対処: **実機でビルド・実行**するか、Xcode > Settings > Platforms で不要なランタイムをさらに削除する。

## シミュレータのランタイムをさらに減らす

Xcode を開く → **Settings (⌘,)** → **Platforms**  
使わない iOS バージョンの「−」で削除すると、そのランタイム用ボリュームが解放される。

## コマンドでランタイム確認

```bash
xcrun simctl runtime list
```
