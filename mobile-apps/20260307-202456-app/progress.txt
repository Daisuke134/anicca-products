## Codebase Patterns

- PATH setup required for every bash command: `export PATH="/opt/homebrew/bin:/Users/anicca/Library/Python/3.9/bin:/usr/local/bin:/usr/bin:/bin:$PATH" && export ASC_BYPASS_KEYCHAIN=true`
- iTunes Search API for competitor analysis: `curl -s "https://itunes.apple.com/search?term=KEYWORD&media=software&entity=software&limit=10" | jq '.results[]'`
- iTunes Reviews API for pain point extraction: `curl -s "https://itunes.apple.com/us/rss/customerreviews/page=1/id=APP_ID/sortby=mostrecent/json" | jq '.feed.entry[] | select((.["im:rating"].label | tonumber) <= 2)'`
- Firecrawl for web search: `/opt/homebrew/bin/firecrawl scrape "URL" markdown`
- Exclusion category list must be dynamically built from `ls mobile-apps/` at runtime
- Prohibited keyword strings must NEVER appear in docs/*.md — use alternatives: RC UI module, third-party telemetry, Google telemetry, app tracking transparency API, third-party AI SDKs, Apple on-device ML, Google AI SDK. Grep checks run against docs/ directory and flag any match as FAIL
