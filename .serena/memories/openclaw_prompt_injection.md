OpenClaw agent prompt/instructions are injected from bootstrap files in agents.defaults.workspace on first turn of new session: AGENTS.md, SOUL.md, TOOLS.md, BOOTSTRAP.md, IDENTITY.md, USER.md. To prevent wrong claims like "browser search unavailable", put tool-availability rules in TOOLS.md/AGENTS.md in the OpenClaw workspace (e.g., ~/.openclaw/workspace/TOOLS.md on Mac Mini) rather than trying to add unsupported keys like agents.list[].systemPrompt in openclaw.json (schema rejects it). web_search/web_fetch/browser are built-in tools; enabling is via tools.profile/tools.allow/deny and tools.web.* config.
